How to use Form.js

The provided form.js script defines a Form class and integrates it into a global namespace F1.lib. This Form class encapsulates all the necessary functionality to handle form interactions, validations, and event handling in a structured manner. 

1
2
3
(function(F1) {
  // Form class definition and other functions here
})(window.F1 = window.F1 || {});

  • The script is wrapped in an IIFE to create a private scope and avoid polluting the global namespace.
  • It accepts F1 as a parameter, ensuring that if F1 already exists, it will be used; otherwise, a new object will be created.

To use the methods from your F1 library classes within your js file, you need to create an instance of the Form class and call the method on that instance. Here's how you can do it, Add this function to the F1.deferred array to initialize the form when the app is ready 

1
2
3
F1.deferred.push(function customFunctionName(app) {
  //methods here
}

After this you need to initialize and load the dependencies in your Js file and use the needed file from the library. for example we are going to use the Form.js

    const Form = F1.lib.Form;
  
    //methods here
  }

validatable()

The validatable function is designed to check if an HTML element should be considered for validation. 

1
2
3
4
5
6
7
function validatable(el) {
  return el.form && el.name && (
    (el.tagName === 'INPUT' && el.type !== 'submit' && el.type !== 'reset') || 
    el.tagName === 'SELECT' || 
    el.tagName === 'TEXTAREA'
  );
}
  • The "el.form" checks if the element is associated with a form. If el.form is truthy, it means the element is part of a form.
  • The "el.name" checks if the element has a name attribute. A name attribute is necessary for form elements that need to be validated and submitted.
  • (el.tagName === 'INPUT' && el.type !== 'submit' && el.type !== 'reset')
    • - " el.tagName === 'INPUT' " This checks if the element is an input field.
    - " el.type !== 'submit' && el.type !== 'reset' " This ensures that the input field is not a submit or reset button. These types of input fields typically do not require validation.
  • el.tagName === 'SELECT': This checks if the element is a dropdown "select".
  • el.tagName === 'TEXTAREA' : This checks if the element is a text area "textarea".

    • The function returns true if the element:

    • Is part of a form (el.form is truthy).
    • Has a name attribute.

    Is either:

    • An input field (but not a submit or reset button).
    • A dropdown (" select").
    • A text area (" textarea").

    If all these conditions are met, the element is considered validatable.


    Example

    1. Validatable Element:
      2. 1
      <input type="text" name="username" form="userForm">

      This input element will be considered validatable because it is an input field with a name attribute and is part of a form.

    2. Non-Validatable Element:
      3. 1
      <input type="submit" name="submitBtn" form="userForm">

      This input element will not be considered validatable because it is a submit button.

    3. Validatable Element:
      4. 1
      <textarea name="comments" form="feedbackForm"></textarea>

      This textarea element will be considered validatable because it is a text area with a "name" attribute and is part of a form.

    By using this function, you can easily determine which elements in a form should be validated.

    F1.deferred.push(function customFunctionName(app) {
 
    // load the form.js
    const Form = F1.lib.Form;
 
 
   // Function to check if an element is validatable
   function validatable(el) {
     return el.form && el.name && (
       (el.tagName === 'INPUT' && el.type !== 'submit' && el.type !== 'reset') || 
       el.tagName === 'SELECT' || 
       el.tagName === 'TEXTAREA'
     );
   }
 
   // Initialize form validation
   function initializeFormValidation(form) {
     const elements = form.elements;
 
     // Iterate over form elements
     for (let i = 0; i < elements.length; i++) {
       const element = elements[i];
 
       // If element is validatable, add 'change' event listener
       if (validatable(element)) {
         element.addEventListener('change', function() {
 
           // Example validation logic
           console.log(`Validating element: ${element.name}`);
           if (!element.value.trim()) {
             console.error(`Validation failed for ${element.name}`);
           } else {
             console.log(`Validation passed for ${element.name}`);
           }
         });
       }
     }
   }
 
   // Run when DOM is ready
   document.addEventListener('DOMContentLoaded', function() {
     const form = document.querySelector('#yourFormId'); // Replace with your form ID
     if (form) {
       initializeFormValidation(form);
     }
   });
 });

    Explanation

    1. "validatable" function:
      • Checks if an element should be validated.
    2. "initializeFormValidation" function:
      • Iterates over form elements.
      • Adds an event listener for validatable elements to perform validation on change.
    3. "DOMContentLoaded" event listener:
      • Runs when the DOM is fully loaded.
      • Selects the form by its ID and initializes validation.
